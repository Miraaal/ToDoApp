# ToDoApp UI/UX 기획서

> 화면별 UI 요소 및 기능 정의

---

## 핵심 시스템

### 아이템 보상 시스템
루틴 완료 시 난이도에 따라 아이템 획득

**일일 루틴 보상: 메소 (4단계)**
- 1단계: 동전
- 2단계: 금동전
- 3단계: 지폐다발
- 4단계: 보따리

**주간 루틴 보상: 코어 젬스톤 (4단계)**
- 1단계: 흰색 젬스톤
- 2단계: 초록 젬스톤
- 3단계: 빨강 젬스톤
- 4단계: 노랑 젬스톤

**월간 루틴 보상: 솔 에르다 (3단계)**
- 1단계: 소형 솔 에르다
- 2단계: 중형 솔 에르다
- 3단계: 대형 솔 에르다

### 포인트(골드) 시스템
- 아이템을 포인트로 전환 가능 (인벤토리에서 판매)
- 포인트는 상점에서 보상 아이템 구매에 사용

### 상점/인벤토리 시스템
- **인벤토리**: 획득한 아이템 보관
  - 아이템별 보유 개수 표시
  - 아이템 → 포인트 전환 (판매) 기능
  - 사용 이력 기록
- **상점**: 포인트로 보상 구매
  - 행동권(티켓) 구매
  - 보상 아이템 (예: 배달음식 시켜먹기)
  - 구매 후 인벤토리에 보관

## 1. 홈 화면 (신규)

### 상단 영역
- [ ] 환영 메시지 (예: "안녕하세요, [사용자]님!")
- [ ] 기간 선택 토글 버튼
  - [ ] 일간 / 주간 / 월간 전환
  - [ ] 선택된 기간에 따라 하단 모든 정보 동적 변경

### 가계부 요약 카드 (컴팩트)
- [ ] 선택된 기간의 재무 현황
  - [ ] 총 수입 (파란색)
  - [ ] 총 지출 (빨간색)
  - [ ] 수지 차액 (양수: 초록색, 음수: 빨간색)
- [ ] 예산 대비 사용률 (월간 선택 시만 표시)
  - [ ] 프로그레스 바
  - [ ] 잔여 예산 표시
- [ ] 터치 시 가계부 화면으로 이동

### 루틴 완료 현황 카드
- [ ] 선택된 기간의 루틴 달성률
  - [ ] 완료율 퍼센테지 (큰 숫자)
  - [ ] 완료/전체 개수 (예: 5/10 완료)
  - [ ] 원형 또는 선형 프로그레스 바
- [ ] 일간 선택 시: 오늘 완료해야 할 루틴 미리보기 (최대 3개)
- [ ] 터치 시 루틴 화면으로 이동

### 동기부여 섹션
- [ ] 연속 달성 일수 (스트릭)
  - [ ] 🔥 아이콘과 함께 "N일 연속 달성!"
  - [ ] 최장 기록 표시 (예: "최고 기록: 30일")
- [ ] 오늘 획득 가능한 아이템 미리보기
  - [ ] 미완료 루틴의 난이도별 아이템 아이콘 표시
  - [ ] "오늘 N개의 아이템 획득 가능!" 메시지
- [ ] 레벨/경험치 시스템 (선택적, 미래 확장)
  - [ ] 현재 레벨
  - [ ] 다음 레벨까지 경험치 바
  - [ ] 필요 경험치 표시

### 획득 아이템 요약 (신규)
- [ ] 인벤토리 미리보기
  - [ ] 주요 아이템 3-4개 표시 (메소/젬스톤/솔 에르다)
  - [ ] 각 아이템별 보유 개수
  - [ ] "인벤토리 전체보기" 버튼
- [ ] 포인트 전환 제안
  - [ ] "N개 아이템을 포인트로 전환 가능" 알림
  - [ ] "빠른 전환" 버튼 (인벤토리로 이동)

### 보유 포인트 & 추천 보상
- [ ] 현재 보유 포인트 강조 표시
  - [ ] 골드 아이콘과 함께 큰 숫자
- [ ] 구매 가능한 추천 보상 아이템 (1-2개)
  - [ ] 아이템 이미지/아이콘
  - [ ] 아이템 이름 (예: "배달음식 시켜먹기")
  - [ ] 필요 포인트
  - [ ] "구매하기" 버튼
- [ ] 터치 시 상점 화면으로 이동

### 빠른 액션 영역
- [ ] 주요 액션 버튼 (2-3개)
  - [ ] "루틴 체크하기" 버튼 → 루틴 화면 이동
  - [ ] "지출 추가하기" 버튼 → 거래 추가 다이얼로그
  - [ ] "통계 보기" 버튼 → 통계 화면 이동 (선택적)

### 기타
- [ ] 전체 영역 스크롤 가능
- [ ] Pull-to-refresh로 데이터 새로고침
