# ToDoApp 기획서

> Unity 기반 루틴 관리 애플리케이션 기능 명세

## 5. 데이터 구조

### 5.1 routines 테이블
| 필드 | 타입 | 설명 | 필수 |
|------|------|------|------|
| id | INTEGER | 고유 ID (자동증가) | ✓ |
| title | TEXT | 루틴 제목 | ✓ |
| type | TEXT | 루틴 타입 (daily/weekly/monthly) | ✓ |
| category | TEXT | 카테고리 (운동/공부/건강/취미) | |
| description | TEXT | 설명 | |
| is_active | INTEGER | 활성 상태 (0/1) | ✓ |
| created_at | TEXT | 생성 시간 (KST) | ✓ |
| updated_at | TEXT | 수정 시간 (KST) | ✓ |

### 5.2 routine_completions 테이블
| 필드 | 타입 | 설명 | 필수 |
|------|------|------|------|
| id | INTEGER | 고유 ID (자동증가) | ✓ |
| routine_id | INTEGER | 루틴 ID (외래키) | ✓ |
| start_date | TEXT | 기준 날짜 | ✓ |
| is_completed | INTEGER | 완료 여부 (0/1) | ✓ |
| completed_at | TEXT | 완료 시간 (KST) | |
| updated_at | TEXT | 수정 시간 (KST) | ✓ |

### 5.3 transactions 테이블 (가계부)
| 필드 | 타입 | 설명 | 필수 |
|------|------|------|------|
| id | INTEGER | 고유 ID (자동증가) | ✓ |
| type | TEXT | 거래 타입 (income/expense) | ✓ |
| amount | INTEGER | 금액 | ✓ |
| category | TEXT | 카테고리 (식비/교통/쇼핑/급여 등) | ✓ |
| description | TEXT | 내용 설명 | |
| transaction_date | TEXT | 거래 날짜 (yyyy-MM-dd) | ✓ |
| created_at | TEXT | 생성 시간 (KST) | ✓ |
| updated_at | TEXT | 수정 시간 (KST) | ✓ |

### 5.4 budget 테이블 (예산 관리)
| 필드 | 타입 | 설명 | 필수 |
|------|------|------|------|
| id | INTEGER | 고유 ID (자동증가) | ✓ |
| month | TEXT | 해당 월 (yyyy-MM) | ✓ |
| category | TEXT | 카테고리 | ✓ |
| budget_amount | INTEGER | 예산 금액 | ✓ |
| created_at | TEXT | 생성 시간 (KST) | ✓ |
| updated_at | TEXT | 수정 시간 (KST) | ✓ |

## 7. 기술적 고려사항

### 7.1 성능
- UniTask를 통한 비동기 처리
- 루틴 목록 동적 생성 최적화
- DB 쿼리 효율화

### 7.2 안전성
- SQL Injection 방지 (파라미터 바인딩)
- 트랜잭션을 통한 데이터 무결성 보장
- UI-DB 동기화 실패 시 롤백

### 7.3 플랫폼 대응
- Windows: StreamingAssets 읽기 전용 처리
- Android: APK 내 DB를 내부 저장소로 복사
- 한국 시간(KST, UTC+9) 일관성 유지
